# 🧩 Lean Canvas Web App

> 비즈니스 모델을 코드로 구조화하는 프론트엔드 프로젝트  
> 기능 구현 이전에 *문제·가설·검증*을 먼저 설계하는 개발 연습

---

## 🧠 프로젝트 소개

린 캔버스(Lean Canvas)는 스타트업과 신생 기업이  
비즈니스 모델을 빠르게 정리하고, 가설을 검증하기 위해 사용하는  
한 장짜리 사업 모델 프레임워크입니다.

이 프로젝트는 단순히 React 컴포넌트를 만드는 데서 끝나지 않고,

- 왜 이 서비스가 필요한지
- 어떤 문제를 어떤 고객을 위해 해결하는지
- 가설을 어떻게 구조화하고, 어떻게 검증할 것인지

를 프론트엔드 애플리케이션 구조로 명확히 표현하는 것을 목표로 합니다.

> “무엇을 만들었는가”보다  
> “왜 그렇게 설계했는가”가 드러나는 코드를 지향합니다.

---

## 🎯 프로젝트 목적

- Todo 리스트를 넘어서, 실제 비즈니스 맥락을 가진 웹 서비스 설계 경험
- 문제 정의 → 해결책 → 검증 → 개선으로 이어지는 반복 가능한 구조 구현
- 프론트엔드 개발자가  
  단순 UI 구현자가 아닌 의사결정에 기여하는 역할을 수행하기 위한 사고 훈련

---

## 💡 왜 린 캔버스인가

린 캔버스는 한 번 작성하고 끝나는 문서가 아니라,

- 가설을 세우고
- 빠르게 검증하며
- 피드백을 통해 지속적으로 수정하는

‘변화하는 사고 도구’입니다.

이 프로젝트에서는 린 캔버스를

- 단순 입력 폼이 아닌
- 사고의 흐름이 드러나는 인터페이스로 구현하고
- 실제 서비스 기획과 유사한 의사결정 구조를 코드로 표현하는 것

에 집중합니다.

---

## 🚀 이 프로젝트를 통해 보여주고 싶은 역량

- 기술 중심 구현에서 문제 중심 설계로 사고 전환
- “이 기능이 왜 필요한가?”를 논리적으로 설명할 수 있는 개발 역량
- 협업·기획·의사결정이 중요한 환경에서도  
  프론트엔드 개발자로 기여할 수 있는 판단력

> 프리랜서 환경에서 요구되는  
> *“빠른 이해 · 명확한 구조 · 설득 가능한 결과물”*을 목표로 합니다.

---

## 🛠 기술 스택

### 현재 사용

- React
- Vite – 빠른 개발 환경 구성
- JavaScript (ES6+)

### 추후 적용 예정 (프로젝트 확장 단계)

- TypeScript – 복잡한 도메인 모델 관리
- Tailwind CSS 또는 CSS-in-JS – UI 구조 실험 및 유지보수성 비교
- Next.js – 라우팅, SEO, 서비스 확장 관점 적용

> 기술은 목적에 따라 선택하며,  
> “배웠기 때문에 쓰는 기술”은 지양합니다.

---

## 🗂 프로젝트 구조 (예정)

src/
├─ components/ # 도메인 단위 UI 컴포넌트
├─ pages/ # 사용자 흐름 중심 화면 구성
├─ hooks/ # 비즈니스 로직을 담은 커스텀 훅
├─ store/ # 상태 및 가설 관리
├─ utils/ # 검증/계산/공통 로직
└─ assets/ # 정적 리소스

---

## 🔮 향후 확장 계획

- 린 캔버스 항목별 가설 관리
- 검증 상태 흐름 (가설 → 실험 → 결과 → 개선)
- 사용자 피드백 히스토리 추적
- 실제 서비스 기획 문서와 연결 가능한 구조
